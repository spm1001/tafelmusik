Architectural rethink session (2026-04-01 evening through 2026-04-02 morning). The accumulated brittleness of the current stack (y-websocket monkey-patched, pycrdt-store squashing disabled, signal files as notification workaround, drift tracking) prompted a full reassessment. Key conclusions:

Comments are messages, not annotations. The system is a messaging layer with one novel property: content-addressed anchoring into shared artifacts. Everything else (rooms, threading, persistence, delivery) is solved messaging infrastructure. The ASGI server already IS a lightweight message broker — it has rooms, WebSocket connections, broadcast. Adding a comments table to SQLite and a second message type on the existing WebSocket is the thinnest viable path.

Comments are standalone entities in SQLite, not Y.Map entries in the CRDT. This decouples comment lifecycle from document lifecycle — comments survive flush, restart, CRDT state reset. The anchoring uses TextQuoteSelector (W3C Web Annotation): quoted text + 30 chars prefix/suffix context. Four-strategy re-anchoring cascade: exact match → disambiguation by context → fuzzy match (SequenceMatcher) → context recovery (find prefix...suffix region when quote is deleted). Comments that can't re-anchor become orphaned but are never deleted — their quoted text carries context even without a position.

Yjs stays for the document sync layer. Deep Research found that CM6's OT collab doesn't guarantee position convergence across peers (Marijn's own caveat), making it worse for annotation anchoring, not better. The Remirror team's failure with external position-based annotation doesn't apply to content-based anchoring because text search re-finds from current state regardless of sync latency. The "keep CRDT for text, SQLite for comments" split gives convergent concurrent editing where needed (text) and lifecycle independence where needed (comments).

The schema is deliberately minimal: id, author, created, target, body, quote, prefix, suffix, replies_to, resolved. Nine fields. No motivation taxonomy, no structured payloads — both parties (human and LLM) can parse natural language. The target is an opaque string the system doesn't interpret; surfaces resolve it. This means the same comment system works for documents, conversations, source files, bon items, URLs.

Built and tested: `anchored.py` (200 lines, 28 tests), `playground.py` (interactive CLI for playing with comments on real files), tmux integration (Ctrl-b C popup for quick reactions anchored to selected text). The tmux flow is: select text → popup → type reaction → stored in SQLite → Claude reads with `read-comments.py`. This eliminated the manual serialisation loop (copy from TUI → paste in editor → structure → paste back).
