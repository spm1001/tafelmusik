## Gotchas
- `_write_signal()` produces duplicate entries per comment (tfm-roturo) — debouncer not catching Y.Map per-field observer fires
- `flush_doc` doubles the room path — room `batterie/tafelmusik/docs/logging` writes to `docs/batterie/tafelmusik/docs/logging.md` (tfm-rimuno)
- FileChanged hooks are user-notification-only — `systemMessage` goes to transient UI widget, NOT model context. Dead end for push notifications.
- `wsLastMessageReceived` uses `Date.now()` (milliseconds), not seconds — got this wrong initially and made blips worse
- systemd unit file (`~/.config/systemd/user/tafelmusik.service`) was modified to add ws-ping-interval/timeout — not version-controlled

## Risks
- y-websocket monkey-patch (`wsLastMessageReceived = Date.now()` every 10s) is fragile — breaks silently on library update. Filed tfm-salima to replace y-websocket entirely.
- Background task mousetrap is one-shot — only first comment gets push delivery per restart. Filed tfm-gesegu for auto-reset.
- asyncRewake hook was removed from settings.json (spike cleanup) — next session needs to decide whether to put it back or use mousetrap only
