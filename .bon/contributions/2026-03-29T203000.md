Transaction origins in pycrdt are local to a Doc instance — they don't survive binary encoding over the wire. This was validated empirically (provenance_test.py). The practical consequence: the change observer that filters Claude's own edits MUST live inside the MCP server process (same Doc), not in a separate channel server process. The original three-process architecture (ASGI + MCP + channel) collapsed to two processes because of this constraint. Formatting attributes (attrs on text.insert) DO survive the wire and persistence, making per-character authorship tracking viable. The combination — origins for filtering, attrs for capture — means the MCP server is becoming "Claude's brain": not just tools, but awareness of who wrote what and when the document changes.
