## Gotchas
- `logging.basicConfig(level=logging.INFO)` in asgi_server.py is a uvicorn workaround — uvicorn configures its own loggers but not root, so our getLogger(__name__) calls need a root handler
- `receive.denyCurrentBranch = updateInstead` is a git config on hezza, NOT version-controlled. Must be re-set after re-cloning. Documented in CLAUDE.md.
- `Room._run()` awaits `_store.db_initialized.wait()` before setting ready — SQLiteYStore.__aenter__ returns before DB init completes (uses start_soon internally)
- Deploy hooks are symlinked: `.git/hooks/post-{commit,merge,receive}` → `../../deploy/*.sh`. Relative paths, must be re-created after re-cloning.
- Mac needs `git remote add hezza modha@hezza:Repos/batterie/tafelmusik` for push-to-deploy (one-time, already done)

## Risks
- Room cleanup is instant (no grace period). A solo editor who refreshes gets room destroyed + SQLite restore. Fast but adds a beat of latency. Could add a timer delay later.
- Post-commit hook fires on every commit touching src/ — during sessions with many small commits, the service restarts repeatedly. Each restart is ~2s and clients reconnect, but it's noisy.
- No rollback mechanism if bad code deploys. Only fix is another push.
