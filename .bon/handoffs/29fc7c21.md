## Gotchas
- `_extract_sections` doesn't respect heading hierarchy — treats ALL headings as section boundaries. Notifications are subsection-granular, which is informative but inconsistent with how `find_section` works (level-aware). Not wrong, but a design choice nobody made consciously.
- `_extract_sections` silently loses duplicate headings (dict key collision). Unlikely in practice.
- Observer calls `str(text)` inside a synchronous pycrdt observe callback. Works because nested `doc.transaction()` reuses the existing ReadTransaction. Undocumented pycrdt behaviour — no explicit test for it.
- Debounce tests add ~12s to the test suite (4 tests × 3s waits). `_change_consumer` accepts a `debounce` parameter but `connect()` doesn't expose it. Threading it through would let tests use 0.1s.
- `pending_notifications` is an unbounded list — nothing consumes it yet. tfm-decigu will wire Channel delivery.

## Risks
- Observer callback runs inside pycrdt's Rust transaction boundary. The try/except prevents crashes, but a broken observer would silently stop notifications without any visible error.
- The debounce timer uses `asyncio.wait_for(queue.get(), timeout=debounce)`. Safe on Python 3.14 but older Pythons had bugs with wait_for task cancellation.
