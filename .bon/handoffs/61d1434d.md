## Gotchas
- Base Provider has a liveness bug: `_run()` exits on disconnect but `_send_updates()` hangs forever. SyncAwareProvider cancels the task group scope when `_run()` exits to fix this.
- `aconnect_ws` cancel scopes are task-bound. Pytest async generator fixtures may run teardown in a different task. Use `@asynccontextmanager` helpers instead (see `make_state` in mcp_server_test.py).
- `RoomConnection.close()` catches RuntimeError from `_exit_stack.aclose()` — dead connections' cancel scopes may fail during cleanup.
- httpx-ws requires `http://` not `ws://` for WebSocket upgrade. `TAFELMUSIK_URL` env var uses `ws://`; converted in `AppState.connect()`.

## Risks
- SyncAwareProvider overrides `_run()` — if pycrdt upstream changes that method, our override must track it. Currently matches pycrdt 0.12.50.
- Test port allocation via `get_free_port()` has a TOCTOU window (port freed before uvicorn binds). Rare but possible in CI.
