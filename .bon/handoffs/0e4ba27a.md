## Gotchas
- `dead.is_set()` must be checked BEFORE `task.cancel()` — cancellation triggers the finally block which always sets dead
- Starlette WebSocket state machine has two failure modes: WebSocketDisconnect on first failed send, RuntimeError on subsequent sends. Must catch both.
- `asyncio.wait()` with `return_when=FIRST_COMPLETED` needed for racing synced vs dead events
- The systemd unit is NOT auto-restarted on code changes — manual `systemctl --user restart tafelmusik` needed
