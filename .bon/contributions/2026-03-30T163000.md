# Calute Phase 1: Implementation Brief

This is a concrete implementation guide for the next Claude session. The design was worked out discursively in the session that produced this file — all decisions below are resolved and agreed with Sameer.

## What calute IS

The .md file is truth. The CRDT is the collaboration overlay. Hydrate CRDT from file on room open, flush CRDT to file explicitly when done. SQLite holds Yjs updates only during live sessions. Comments wipe on flush. The flush IS the compaction — sidesteps the pycrdt-store squashing bug.

## Phase 1 scope (what to build first)

### 1. Docs anchor directory

The ASGI server needs a `docs_dir` parameter — a directory of .md files it serves. Default: `ROOT / "docs"` (alongside existing `data/` and `public/`).

In `create_app()` in `asgi_server.py`, add `docs_dir: str | Path = ROOT / "docs"` parameter.

### 2. Room name = relative file path

Room names become file paths relative to `docs_dir`. Room `meeting/2026-03-30` maps to `docs/meeting/2026-03-30.md`. The browser URL becomes `?room=meeting/2026-03-30`.

The existing `/{room:path}` WebSocket route already handles path-style room names. No route changes needed.

### 3. Hydrate from file on room open

Change `RoomManager.get_room()`:

```python
async def get_room(self, name: str) -> Room:
    if name not in self.rooms:
        doc = Doc()
        text = doc.get("content", type=Text)

        # Try to hydrate from .md file first
        md_path = self._docs_dir / f"{name}.md"
        if md_path.exists():
            content = md_path.read_text()
            with doc.transaction():
                text += content
            log.info("Room %s: hydrated from %s", name, md_path)
        else:
            # Fall back to SQLite restore (backwards compat during migration)
            doc = await _restore_ydoc(self._store_cls, name)
            log.info("Room %s: restored from store (no .md file)", name)

        store = self._store_cls(path=name)
        room = Room(name, doc, store)
        await room.start()
        self.rooms[name] = room
    return self.rooms[name]
```

Key: `text += content` is equivalent to `text.insert(0, content)` — it sets the initial content. No authorship attrs on hydrated content (it's the file's content, not anyone's edit).

### 4. flush_doc MCP tool

New tool in `mcp_server.py`:

```python
@mcp.tool()
async def flush_doc(room: str, ctx: Context) -> str:
    """Flush document to .md file on disk, wipe comments, discard CRDT log.

    This is the 'save' — writes current Y.Text content to the .md file,
    clears all comments from Y.Map, and the CRDT update history can be
    discarded (fresh hydrate on next room open).
    """
    state = _get_state(ctx)
    conn = await _get_or_create_connection(state, room)

    text = conn.doc.get("content", type=Text)
    content = str(text)

    # Write to file
    docs_dir = Path(os.environ.get("TAFELMUSIK_DOCS_DIR", "docs"))
    md_path = docs_dir / f"{room}.md"
    md_path.parent.mkdir(parents=True, exist_ok=True)
    md_path.write_text(content)

    # Wipe comments
    comments_map = conn.doc.get("comments", type=Map)
    with conn.doc.transaction(origin=authors.CLAUDE):
        for key in list(comments_map):
            del comments_map[key]

    # Git commit
    # (subprocess call to git add + git commit)

    return f"Flushed {len(content)} chars to {md_path}. Comments cleared."
```

The `docs_dir` needs to be discoverable by both processes. Options: env var `TAFELMUSIK_DOCS_DIR`, or derive from `TAFELMUSIK_URL` + an API call. Env var is simpler.

### 5. Directory listing replaces room-based list_docs

Change `list_rooms()` in `asgi_server.py`:

```python
async def list_rooms(request):
    memory_rooms = set(manager.rooms.keys())
    persisted_rooms = await asyncio.to_thread(_query_persisted_rooms)

    # Also list .md files in docs_dir
    file_rooms = set()
    if docs_dir.exists():
        for md_file in docs_dir.rglob("*.md"):
            rel = md_file.relative_to(docs_dir).with_suffix("")
            file_rooms.add(str(rel))

    all_rooms = sorted(memory_rooms | persisted_rooms | file_rooms)
    return JSONResponse({"rooms": all_rooms})
```

### 6. New doc flow

No code change needed for creation — browser navigates to `?room=new/doc/name`, CRDT room is created empty (no .md file to hydrate from), user types, Claude edits, then `flush_doc` writes the .md file to `docs/new/doc/name.md`.

## New tests needed

- Hydrate from .md file: room opens with file content in Y.Text
- Hydrate fallback: room with no .md file falls back to SQLite
- Flush: Y.Text content written to .md, comments wiped
- Directory listing: .md files appear in list_rooms
- New doc flow: flush creates new .md file with parent dirs
- Round-trip: hydrate → edit → flush → hydrate again = same content

## Gotchas the next Claude should know

- `text += content` on an empty Text works fine. On a non-empty Text it appends. For hydrate, the Doc/Text must be fresh (no prior content).
- When creating nested Y.Maps (e.g., comments), integrate into parent FIRST, then set properties: `comments_map[id] = inner_map` before `inner_map["key"] = value`. Reversed order raises `RuntimeError: Not integrated in a document yet`.
- The `_restore_ydoc` function creates a fresh Doc internally. If we switch to hydrate-from-file, we create the Doc ourselves and skip `_restore_ydoc`. Keep `_restore_ydoc` for backwards compat during migration.
- Room names with `/` in them work — the WebSocket route is `/{room:path}` which matches slashes. But the SQLiteYStore `path` parameter uses the room name as a key, not a file path, so slashes are fine there too.
- `md_path.parent.mkdir(parents=True, exist_ok=True)` is needed for nested room paths like `meeting/2026-03-30`.
- The flush git commit should use `git add <specific file>` not `git add .` — avoid accidentally staging other changes.
- Comments wipe is `del comments_map[key]` inside a transaction with Claude as origin. This triggers the observer — the MCP server will see its own comment deletions but filter them via `txn.origin`.
- During migration: some rooms will have SQLite state but no .md file. The hydrate logic should try file first, fall back to SQLite. After migration, SQLite becomes optional.
