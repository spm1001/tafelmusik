## Gotchas
- The overlap-based sort comparator is non-transitive. With many adjacent comments, card ordering could be non-deterministic. Filed as tfm-mabufa.
- marked is still in public/package.json but no longer imported or bundled. Cosmetic — npm uninstall marked when convenient.
- The compose editor uses minimalSetup (not basicSetup) — editing affordances differ slightly from the document editor. Deliberate: comments are short.
- add_comment uses Assoc.AFTER for StickyIndex which can resolve 1 character off from browser-created anchors. The overlap sort compensates, but a future anchor normalisation pass would be cleaner.
- .hypothesis/ directory appeared (from property-based tests in previous sessions). Not tracked in git.

## Risks
- Re-anchoring is not implemented. replace_section (Claude's primary editing mode) destroys comment anchors. Comments fall back to quote search, which works if the text survived verbatim. If rewritten, comments vanish silently. This is the most likely production failure. Filed as tfm-mafuze.
- renderComments() rebuilds the entire DOM on every change. Fine at 10 comments, will flicker/reset scroll at 50+.
- No frontend tests. The state machine and sort logic are complex enough to warrant them. Filed as tfm-gosifa.
