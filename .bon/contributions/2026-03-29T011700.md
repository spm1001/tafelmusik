Tafelmusik's architecture emerged from a specific failure: BlockNote (Y.XmlFragment) looked great for the human but was structurally hostile to Claude — pycrdt can't safely write nested XML trees. The pivot to Y.Text (a plain markdown string) made Claude a true Yjs peer. This is the core architectural insight: choose data structures that both participants can operate on natively. Markdown is what Claude thinks in; Y.Text is markdown as a CRDT. Every other decision flows from this.

The spike (2026-03-29) validated the connection pattern but also revealed API landmines: `WebsocketServer.start()` blocks forever and must be run as `asyncio.create_task()`, `HttpxWebsocket` wraps an existing httpx-ws connection rather than creating one, and `get_room()` is a coroutine. These are the kind of things documentation doesn't tell you and reviews can't catch — only running code reveals them. The spike results are captured as Gotchas 16-17 in the plan.

The comment system uses point+quote anchoring (single StickyIndex + stored quote text) rather than start/end ranges. This came from a user question about whether point insertions would be simpler. They are — and they're more resilient to `replace_section` because there's only one anchor to lose (not two), and the quote string provides a search target for re-anchoring. The design doc's principle 5 still describes the old range-based approach and should be updated when comments are implemented.
