## For Claudes to come

Python's `with sqlite3.connect(path) as conn:` is a transaction manager, not a resource manager — `__exit__` commits or rolls back but does NOT close the connection. This is the most common stdlib footgun and it caused a production outage here: one leaked FD per call, FD limit exhausted in 10 minutes. Always use `try/finally: conn.close()`. The broader lesson: when using any database library's context manager, check what `__exit__` actually does. Don't assume it closes the connection.

When a server is misbehaving, start with system-level observables (`/proc/PID/fd`, `lsof`, `ss`, `ps aux`) before building instrumentation. The kernel already has the telemetry you need — you just need to read it.
